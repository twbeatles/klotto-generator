## 이번에 반영된 핵심 수정

### 1. 당첨 데이터 저장/동기화 경로 통합
- 당첨 데이터 저장 경로를 회차 단위 upsert로 통일
- 회차 메타데이터(`first_prize`, `first_winners`, `total_sales`) 보존
- `0`/`None` 입력이 기존 non-zero 메타데이터를 덮어쓰지 않도록 보정

### 2. 백그라운드 동기화 보강
- 앱 시작 시 최신 회차만 따라가지 않고 최근 20회 구간의 결손 회차를 먼저 보수
- 실패 회차 목록을 로그로 남기도록 개선

### 3. 당첨 정보 위젯/통계 UI 일관성
- 당첨 정보 위젯이 DB 캐시를 먼저 즉시 표시
- 이어서 네트워크로 최신 정보를 확인하고 결과를 DB/통계에 반영
- 실제 당첨 통계 창에서 최근 5회 동기화 후 열린 화면이 즉시 갱신되도록 개선

### 4. 생성 검증 강화
- 홀짝 균형을 만족할 수 없는 고정수/제외수 조합은 생성 전에 즉시 차단
- 후보 고갈을 명시적 실패 사유로 집계
- 생성 실패/부분 성공 시 주요 실패 원인을 함께 안내

### 5. 히스토리/당첨 확인 접근 범위 확장
- 히스토리 창이 저장된 전체 히스토리를 표시
- 당첨 확인 창이 즐겨찾기/히스토리 전체 데이터를 조회 대상으로 사용
- 표시 리스트와 실제 참조 데이터 인덱스를 공통 source list로 정렬
